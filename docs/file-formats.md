# File Formats

Dip-C uses several plain-text file formats, all tab- or comma-delimited. Most can be gzip-compressed (`.gz`) and Dip-C will decompress them transparently.

## Phased SNPs (`.txt`)

A list of phased single-nucleotide polymorphisms (SNPs) in tab-delimited format:

| Column | Description |
|--------|-------------|
| 1 | Chromosome |
| 2 | Coordinate |
| 3 | Paternal nucleotide |
| 4 | Maternal nucleotide |

An example SNP file for the GM12878 cell line (in hg19 coordinates), based on the 2016-1.0 release of [Illumina Platinum Genomes](https://www.illumina.com/platinumgenomes.html), is provided as `snps/NA12878.txt.gz`:

```
1	534324	G	T
1	550515	T	C
1	565419	G	C
1	655642	G	A
1	666543	G	A
```

## Read Segments (`.seg`)

The basic output of a chromatin conformation capture (3C/Hi-C) experiment is sequencing reads containing more than one genomic segment. In each of these *chimeric* reads, genomic segments far away in the linear genome (or from different chromosomes) are joined by proximity-based ligation.

Each line represents a read in tab-delimited format:

| Field | Description |
|-------|-------------|
| 1 | Read name |
| 2+ | Segments (one per field) |

Each segment is a comma-delimited string with 8 fields:

| Field | Description |
|-------|-------------|
| 1 | `.` for read 1, `m` for read 2 (mate) |
| 2 | Start coordinate in the read |
| 3 | End coordinate in the read |
| 4 | Chromosome |
| 5 | Start coordinate in the genome |
| 6 | End coordinate in the genome |
| 7 | Strand (`+` or `-`) |
| 8 | Haplotype (`.` unknown, `0` paternal, `1` maternal) |

Example:

```
HWI-D00433:595:HLYW7BCXY:1:1209:15116:100489	.,0,211,5,94770308,94770519,+,.	m,57,211,5,97167374,97167528,-,0	m,0,61,5,94770541,94770602,-,.
HWI-D00433:595:HLYW7BCXY:1:1113:20520:48210	.,0,193,21,23683758,23683951,-,0	.,188,255,21,25124149,25124216,-,.
```

Generated by:

```sh
dip-c seg input.bam > output.seg
dip-c seg -v snps.txt input.bam > output.phased.seg   # with haplotype phasing
```

## Contact Legs (`.leg`)

A "leg" is an endpoint of a read segment — it forms one of the two legs of a chromatin contact. More generally, a leg can be any single point in the genome.

Each leg is a comma-delimited string: chromosome, coordinate, haplotype.

```
1,948359,.
1,1192624,.
1,2561820,.
1,2954969,1
```

## Contacts (`.con`)

A contact refers to the proximity-based ligation of two genomic segments. A contact is defined as the two adjoining endpoints (legs) of two different segments in the same read.

Each line is tab-delimited: leg 1, leg 2.

```
1,858641,.	1,861338,.
1,861471,.	1,862872,.
1,918037,.	1,1024147,.
1,918249,1	1,1231502,.
```

Generated by:

```sh
dip-c con output.seg > output.con
```

!!! note "Subtleties"
    - When a read contains more than two segments, contacts are defined between **all** pairs of segments.
    - The two legs of a contact are interchangeable; leg 1 < leg 2 by convention.
    - Directionalities are ignored. [hickit](https://github.com/lh3/hickit) preserves directionality via the [`.pairs` format](https://github.com/4dn-dcic/pairix/blob/master/pairs_format_specification.md).

## 3D Genomes (`.3dg`)

The primary output of the Dip-C algorithm: the 3D structure of a single-cell genome. Following [nuc_dynamics](https://github.com/TheLaueLab/nuc_dynamics), each chromosome is represented by 3D coordinates of regularly spaced points (0 kb, 10 kb, 20 kb, 30 kb ...) along the chromosome.

Tab-delimited format:

| Column | Description |
|--------|-------------|
| 1 | Chromosome with haplotype: `1(pat)`, `1(mat)` |
| 2 | Coordinate |
| 3 | x |
| 4 | y |
| 5 | z |

```
1(mat)	1420000	0.791377837067	10.9947291355	-13.1882897693
1(mat)	1440000	-0.268241283699	10.5200875887	-13.0896257278
1(mat)	1460000	-1.3853075236	10.5513787498	-13.1440142173
```

## Genomic Regions (`.reg`)

Similar to a [BED file](https://genome.ucsc.edu/FAQ/FAQformat.html#format1), but with haplotype information. Used to exclude copy-number gains, set haplotypes in CN-loss regions, or extract regions of interest.

Tab-delimited format:

| Column | Description |
|--------|-------------|
| 1 | Chromosome |
| 2 | Haplotype (`.` for both) |
| 3 | Start coordinate (`.` for chromosome start) |
| 4 | End coordinate (`.` for chromosome end) |

```
1	.	.	.
2	1	232800000	.
5	.	165800000	167800000
6	1	.	.
```

## Color Files (`.color`)

Color files assign a numeric value to each particle in a 3D genome. Tab-delimited: chromosome with haplotype, coordinate, value.

```
1(pat)	20000	0.015
1(pat)	40000	0.008
1(pat)	60000	0.012
```

Generated by `dip-c color` and consumed by `dip-c vis -c`.
