# Visualization

## Contact Maps with Juicebox

Convert a `.con` file to Juicebox format:

```sh
scripts/con_to_juicer_pre_short.sh dedup.con.gz
java -Xmx2g -jar juicer_tools.jar pre -n dedup.juicer.txt.gz dedup.hic hg19
```

For haplotype-resolved contacts:

```sh
scripts/con_imputed_to_juicer_pre_short.sh impute3.round4.con.gz
java -Xmx2g -jar juicer_tools.jar pre -n impute3.round4.juicer.txt.gz impute3.round4.hic hg19.chr.hom.len
```

The output `.hic` file can be viewed interactively in [Juicebox](http://www.aidenlab.org/juicebox/) (the error message about normalization can be ignored).

## 3D Genome Visualization with PyMOL

### Setup

Before viewing any mmCIF file, add this line to your PyMOL startup script (`~/.pymolrc`). Without it, PyMOL may create bonds between numerous particle pairs, consuming excessive CPU and memory.

```python
set connect_mode, 4
```

### Color by Chromosome

Color a 3D genome by chromosome (rainbow: blue = chr 1, red = chr X/Y) and convert to mmCIF:

```sh
dip-c color -n hg19.chr.txt cell.3dg | dip-c vis -c /dev/stdin cell.3dg > cell.n.cif
```

!!! note "Bundled data files"
    The file name `hg19.chr.txt` is resolved automatically from the bundled data. You don't need to provide the full path.

Open `cell.n.cif` in PyMOL and style:

```python
viewport 800, 800
set ray_shadows, 0
as sticks, all
set_bond stick_radius, 0.5, all
spectrum b, rainbow, all, 1, 23
png cell.n.png, 800, 800, ray=1
```

![PyMol chromosome](images/pymol.n.png){ width="400" }

### Expand into Separate Chromosomes

Move chromosomes apart for better visualization:

```sh
dip-c exp cell.3dg > cell.exp.3dg 2> cell.exp.py
dip-c color -n hg19.chr.txt cell.exp.3dg | dip-c vis -c /dev/stdin cell.exp.3dg > cell.exp.n.cif
```

![PyMol expanded](images/pymol.exp.n.png){ width="400" }

A rotation movie can be generated by modifying `pymol/pymol_movie_exp.py` with the output from `cell.exp.py`. In PyMOL, export via File > Export Movie As > MPEG, selecting "ffmpeg" as the encoder.

### Label Chromosomes

Store the camera position with `get_view` in PyMOL, then:

```sh
dip-c exp -c cell.3dg > cell.exp_c.3dg
dip-c vis cell.exp_c.3dg | sed 's/(mat)/♀/g; s/(pat)/♂/g' > cell.exp_c.cif
```

Style with the saved camera view and overlay labels:

```python
hide all
label all, chain
set label_size, 10
set label_bg_color, white
set label_bg_transparency, 0.4
```

![PyMol labeled](images/pymol.exp.n.label.png){ width="400" }

### Color by CpG Frequency

Calculate CpG frequency (or use bundled precomputed files):

```sh
scripts/cpg.sh genome.fa 20000 > hg19.cpg.20k.txt
```

Color and visualize:

```sh
dip-c color -c hg19.cpg.20k.txt cell.3dg | dip-c vis -M -c /dev/stdin cell.3dg > cell.cpg.cif
```

Style in PyMOL using the [spectrumany](https://pymolwiki.org/index.php/Spectrumany) plugin to show a single slice:

```python
viewport 800, 800
clip slab, 10
set ray_shadows, 0
set ambient, 1
set specular, off
set ray_opaque_background, off
as sticks, all
set_bond stick_radius, 0.5, all
spectrumany b, magenta green, all, 0.005, 0.02
png cell.cpg.png, 800, 800, ray=1
```

Serial slicing:

```python
clip move, 15
```

![PyMol CpG](images/pymol.cpg.png){ width="400" }

### Haploid Cells

Dip-C can handle haploid cells with minor modifications. A haploid `.3dg` file has chromosome names like `1` instead of `1(mat)` and `1(pat)`. To visualize, append a dummy haplotype (e.g., `(mat)`) to all chromosome names.

For haploid mouse embryonic stem cells (mESCs), `(mat)` is accurate since they are typically generated from oocytes. For other haploid cells (e.g., eHAP), it is only a placeholder.

## Contact Matrices

Convert a `.con` file into a genome-wide contact matrix with `dip-c bincon`:

```sh
# unphased (merged haplotypes)
dip-c bincon -b 5000000 -H -l hg19.chr.len dedup.con.gz > dedup.bincon.txt
dip-c bincon -i -b 5000000 -H -l hg19.chr.len . > dedup.bincon.info

# haplotype-resolved
dip-c bincon -b 5000000 -l hg19.chr.len impute3.round4.con.gz > impute.bincon.txt
dip-c bincon -i -b 5000000 -l hg19.chr.len . > impute.bincon.info
```

The primary output (`*.bincon.txt`) is the contact matrix. The secondary output (`*.bincon.info`) lists the genomic coordinates of bin centers.
